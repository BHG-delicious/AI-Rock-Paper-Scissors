# 🤖 AI 剪刀石頭布 · 競賽版

本專案為 AI 對戰平台，完整實作「五階段競賽規則」，讓兩位 AI 透過人類中介進行高規格猜拳博弈。

## 🎯 核心特色

- **五階段完整流程**：規則宣讀 → 比賽準備 → 比賽中途 → 比賽結果 → 交換復盤
- **固定選手代號**：統一使用「玩家A」與「玩家B」，簡化辨識
- **規律比賽編號**：格式為 `YYMMDD` + 兩位序號（如 `24072101`），每日重置，於比賽準備階段即產生並顯示
- **動態規則載入**：規則內容儲存於 `rules.txt`，可隨時更新
- **策略提示詞顯示對手 AI 模型名稱**：準備階段的提示詞中，對手名稱顯示其 AI 模型（如「DeepSeek」），而非「玩家B」
- **嚴謹的回合控制**：下一回合按鈕在雙方回應解析完成後才啟用，避免錯誤操作
- **明確的操作回饋**：解析回應後會彈出成功/失敗提示，避免重複解析
- **正確的復盤內容**：根據實際勝者與比分生成復盤提示詞，比分順序統一為「玩家A : 玩家B」
- **靈活策略選擇**：每位玩家可選擇「AI 自選策略」或「人類指定策略」
- **心理戰支援**：允許 AI 在 `reason` 中釋放假訊息
- **復盤與交換**：賽後自動生成復盤提示詞，並支援雙方復盤內容交換

## 🚀 如何使用

### 1. 規則宣讀
- 頁面自動從 `rules.txt` 載入規則，若失敗可手動貼入。
- 點擊「複製規則」將規則分別提供給兩位 AI，請它們回覆「我已完整閱讀比賽規則並同意參賽」。
- 勾選確認後進入下一階段。

### 2. 比賽準備
- 頁面頂部顯示已產生的**比賽編號**（例如 `24072101`），所有策略提示詞均使用此編號。
- 分別設定兩位玩家的 AI 模型（下拉選單），策略提示詞會自動使用對方的 AI 模型名稱作為對手稱呼。
- 選擇策略產生方式：
  - **AI 選擇策略**：系統生成提示詞，AI 回覆策略名稱後貼入。
  - **指定策略**：手動輸入策略名稱，並可生成對應的指定策略提示詞。
- 雙方策略名稱填妥後，「開始比賽」按鈕啟用。

### 3. 比賽中途
- 系統自動產生雙方提示詞（含歷史記錄、對手分析、上一回合 JSON）。
- 將提示詞複製給對應 AI，並將 AI 回覆的 JSON 貼回。
- 點擊「解析回應」，系統會立即告知成功與否；雙方都解析成功後，自動判定勝負並顯示結果。
- **下一回合按鈕**只有在雙方解析完成後才會啟用，點擊後自動清空本回合結果並進入下一回合。

### 4. 比賽結果與復盤
- 宣布獲勝者（實際勝者），並產生復盤提示詞（含最後一回合對手 JSON，比分順序統一為 A : B）。
- 複製給雙方 AI，貼上它們的復盤內容並儲存。

### 5. 交換復盤結果
- 系統將雙方復盤內容互相提供給對方（產生交換提示詞）。
- 複製提示詞給對應 AI，完成最終交流。

## 🛠 技術實現

- **前端**：純 HTML5 + CSS3 + JavaScript（ES6）
- **無後端**：所有邏輯於瀏覽器端執行，資料儲存於 `localStorage`
